1.1 meta标签

下面我想要介绍以下meta标签因为该标签我在最初理解的时候感觉比较混乱，查了些文档也有些迷糊，所以想整理如下:

实际上meta标签只有两个属性,分别是http-equiv和name两个属性。

1.1.1 name属性

其中在name中,我们可以这样理解<meta name="属性参数" content="具体描述">其中name有如下参数：

keywords(关键字): 用来告诉搜索引擎你的关键字是什么，关键字的内容即为上面说的content(具体描述)，举个例子: <meta name="keywords" content="HTML,meta,keywords">

description(网站内容的描述): 用于告诉引擎，你网站的主要内容，和上面的没什么区别。举个例子：<mate name="description" content="只是一个HTML的总结">

viewport(移动端窗口):这一段比较复杂，涉及到手机端的样式，如果想详细了解可以网上查询，<meta name="viewport" content="width=device-width, initial-scale=1.0">这段的意思是指在移动端下宽度为设备的宽度，无缩放。

  • 其他属性:如对爬虫，制作软件，版权的信息说明等…

1.1.2 http-equiv属性

 http-equiv属性的作用是能够改变定义了服务器和用户引擎行为的编译。
 同样http-equiv属性的书写格式和name属性格式没什么区别。<meta http-equiv="属性参数" content="具体描述">,其属性参数简单介绍有如下几种：

  • X-UA-Compatible：浏览器的渲染方式。如<meta http-equiv="X-UA-Compatible" content="ie=edge">则是指ie的edge浏览器版本进行渲染
  • refresh：这个属性指定，如果content为一个整数，那么它指定几秒钟后重新刷新页面，如果在该整数后面跟着字符串链接，那么则在数秒后刷新并进入链接地址。
  • Set-Cookie：如果该网页已经过期了，那么本地的cookies也会被自动删除掉。
  • …

4. 实时的云端文稿

5. 离线模式

在网络环境不稳定的情况下记录文字一样很安全！在您写作的时候，如果电脑突然失去网络连接，Cmd Markdown 会智能切换至离线模式，将您后续键入的文字保存在本地，直到网络恢复再将他们传送至云端，即使在网络恢复前关闭浏览器或者电脑，一样没有问题，等到下次开启 Cmd Markdown 的时候，她会提醒您将离线保存的文字传送至云端。简而言之，我们尽最大的努力保障您文字的安全。
